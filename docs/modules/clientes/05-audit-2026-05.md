**`src/app/api/clientes/[id]/historico/route.ts` — linha 114**
```typescript
// Antes
where: { clienteId: id, empresaId: 1 }, // single-tenant: GladPros empresaId=1

// Depois
const EMPRESA_ID = 1 // single-tenant: GladPros
...
where: { clienteId: id, empresaId: EMPRESA_ID },
```

**Motivação:** Eliminar o "número mágico" inline. A constante nomeada deixa a intenção explícita e facilita futura migração para multi-tenant se necessário.

### ⚠️ Warning — `findMany` sem limite em relatorios

**`src/app/(dashboard)/clientes/relatorios/page.tsx` — linhas 46-50**
```typescript
// Antes
prisma.cliente.findMany({
  where: { criadoEm: { gte: sixMonthsAgo } },
  select: { criadoEm: true },
  orderBy: { criadoEm: "asc" },
})

// Depois
prisma.cliente.findMany({
  where: { criadoEm: { gte: sixMonthsAgo } },
  select: { criadoEm: true },
  orderBy: { criadoEm: "asc" },
  take: 1000,
})
```

**Motivação:** Query usada para dados de gráfico (agrupamento por mês). A janela de 6 meses + `select` mínimo (`criadoEm: true`) já limita o payload, mas `take: 1000` previne crescimento ilimitado à medida que a base de clientes escala.
